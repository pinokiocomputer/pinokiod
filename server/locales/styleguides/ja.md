# Japanese UI Style

- Translate from intent and UI context, not English word order.
- Use natural short UI labels. Avoid full polite sentences on buttons unless the English source is also sentence-like.
- For command buttons, prefer object + particle + verb: `チェックポイントフォルダを開く`, not `開く チェックポイント フォルダー`.
- Do not insert spaces between normal Japanese words. Spaces are acceptable around Latin code terms only when readability requires it.
- Use `フォルダ` consistently for folder.
- Keep `Pinokio`, `Git`, `GitHub`, `HTTP`, `HTTPS`, code identifiers, paths, and URLs literal.
- Preserve placeholders exactly: `{name}` stays `{name}`.
- Rephrase around placeholders rather than attaching Japanese particles directly to ambiguous placeholder values when that may read awkwardly.
- Keep destructive actions clear: delete/remove/reset must not sound like refresh.
- Use consistent terms for app, plugin, launcher, task, workspace, checkpoint, and terminal.
