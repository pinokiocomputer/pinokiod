# Indonesian UI Style

- Use natural Indonesian UI phrasing, not literal English fragments.
- Keep command labels concise and direct: `Buka folder`, `Simpan versi`, `Hapus tugas`.
- Avoid overly formal wording unless the source is a warning or legal/security message.
- Borrow common technical terms when they are natural in Indonesian product UI.
- Keep `Pinokio`, `Git`, `GitHub`, `HTTP`, `HTTPS`, code identifiers, paths, and URLs literal.
- Preserve placeholders exactly: `{name}` stays `{name}`.
- Rephrase around placeholders when affixes would make the placeholder awkward.
- Avoid mixing Indonesian and English grammar in the same phrase.
- Keep buttons short; put explanations in helper text.
- Use consistent terms for app, plugin, launcher, task, workspace, and checkpoint.
