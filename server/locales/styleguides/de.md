# German UI Style

- Use natural German UI phrasing, not English word order.
- Prefer concise infinitive command labels: `Ordner √∂ffnen`, `Version speichern`.
- Capitalize nouns.
- Form compounds naturally when they are common and readable; avoid awkward literal noun chains.
- Keep `Pinokio`, `Git`, `GitHub`, `HTTP`, `HTTPS`, code identifiers, paths, and URLs literal.
- Use German quotation and punctuation naturally in full messages.
- Avoid forcing placeholders into German case endings when the placeholder value is unknown; rephrase if needed.
- Preserve placeholders exactly: `{name}` stays `{name}`.
- Translate destructive commands with clear warning strength; do not soften `Reset`, `Delete`, or `Remove`.
- Keep button labels short. Longer explanatory copy belongs in descriptions, dialogs, or helper text.
