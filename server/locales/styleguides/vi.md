# Vietnamese UI Style

- Use natural Vietnamese UI phrasing, not literal English fragments.
- Keep command labels concise and direct: `Mở thư mục`, `Lưu phiên bản`, `Xóa tác vụ`.
- Use natural Vietnamese modifier order; avoid English noun-chain order.
- Keep tone marks correct.
- Borrow common technical terms when they are natural in Vietnamese product UI.
- Keep `Pinokio`, `Git`, `GitHub`, `HTTP`, `HTTPS`, code identifiers, paths, and URLs literal.
- Preserve placeholders exactly: `{name}` stays `{name}`.
- Rephrase around placeholders when a classifier or modifier would be awkward.
- Keep buttons short; put explanations in helper text.
- Use consistent terms for app, plugin, launcher, task, workspace, and checkpoint.
