# Turkish UI Style

- Use natural Turkish UI phrasing, not English word order.
- Prefer concise command labels with natural object/action order: `Klasörü aç`, `Sürümü kaydet`.
- Turkish suffixes can make placeholders fragile. Avoid attaching suffixes directly to placeholders unless the form is safe for any value.
- Rephrase around placeholders when case, possession, or vowel harmony would be unknown.
- Keep `Pinokio`, `Git`, `GitHub`, `HTTP`, `HTTPS`, code identifiers, paths, and URLs literal.
- Preserve placeholders exactly: `{name}` stays `{name}`.
- Keep button labels short and direct.
- Use clear wording for destructive actions such as delete, remove, and reset.
- Avoid half-English grammar with Turkish words.
- Use consistent terms for app, plugin, launcher, task, workspace, and checkpoint.
