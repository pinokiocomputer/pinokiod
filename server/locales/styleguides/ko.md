# Korean UI Style

- Translate from intent and UI context, not English word order.
- For command buttons, prefer object + action: `체크포인트 폴더 열기`, not `열기 체크포인트 폴더`.
- Keep button labels short and noun/action oriented. Avoid full polite sentences on buttons.
- Use natural Korean spacing with loanwords. Do not mechanically space every English source word.
- Keep `Pinokio`, `Git`, `GitHub`, `HTTP`, `HTTPS`, code identifiers, paths, and URLs literal.
- Preserve placeholders exactly: `{name}` stays `{name}`.
- Rephrase around placeholders when particles such as `을/를`, `이/가`, or `은/는` would be uncertain.
- Keep destructive actions clear: delete/remove/reset must not sound like refresh.
- Avoid half-English grammar with Korean words.
- Use consistent terms for app, plugin, launcher, task, workspace, checkpoint, and terminal.
